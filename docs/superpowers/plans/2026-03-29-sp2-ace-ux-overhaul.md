# SP2 — ACE UX/UI Overhaul Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restructure the ACE frontend to match the design spec: nav rename/cleanup, master-detail assistant editor with five sub-tabs (Overview / Instructions / Skills / Knowledge / Quick Actions), type badges, helper text, and empty states.

**Architecture:** All changes are frontend-only — `admin-editor/public/` is the source of truth; `dist/` is generated by `npm run ace:build`. No new backend routes, no data model schema changes. New fields written to the assistant object (`defaultKbName`, per-QA `kbName`) are additive and handled by the existing POST /api/save flow. The sub-tab state lives in `state.selectedAssistantDetailTab` (in-memory only, not persisted).

**Tech Stack:** Vanilla JS (`admin-editor/public/app.js`), HTML (`admin-editor/public/index.html`), CSS (`admin-editor/public/styles.css`). Local dev server: `npm run admin` (port 3333). Build: `npm run ace:build` copies `public/` → `dist/`.
