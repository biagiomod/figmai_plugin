# Evergreens Module

**Owner:** Evergreens Team  
**Assistant ID:** `content_table`  
**Status:** Live (production)

---

## What This Is

The Evergreens module is the production implementation of the `content_table` assistant. It is a custom build that overrides the default content table handler with its own handler and knowledge base.

This directory is the Evergreens team's local workspace inside the plugin.

**Do not import from `src/core` internals directly** — use `../../sdk` only (see `index.ts`).

---

## Files

| File | Purpose |
|------|---------|
| `index.ts` | Module entry point. Registers this handler for `assistantId: 'content_table'`. |
| `handler.ts` | `ContentTableHandler` — scans selected containers, creates and appends content table rows in Figma. Supports `generate-table` and `add-to-table` actions. |
| `knowledge.md` | Knowledge base content for the Evergreens assistant persona. This is the base prompt template used when the `content_table` assistant is active. |
| `evergreens.generated.ts` | Generated by `scripts/build-assistants.ts`. **Do not edit manually.** |

---

## How It Fits the Plugin

The Evergreens module is loaded as an `AssistantModule`:

```
src/assistants/evergreens/index.ts
  → AssistantModule { assistantId: 'content_table', handler: ContentTableHandler }
  → registered at build time via src/assistants/evergreens/evergreens.generated.ts
```

The handler overrides the handler that would otherwise match `content_table` in `src/core/assistants/handlers/`. The legacy `contentTable.ts` in that directory is **not used in production**.

## Testing Changes Safely

1. `npm run build` — confirm no TypeScript errors
2. `npm run test` — run unit tests
3. `npm run invariants` — confirm plugin invariants still hold
4. In Figma: select a frame and run **Generate Table** or **Add to Table** to verify the content scan output
