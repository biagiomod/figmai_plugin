Design Section 2 of 4 — The SDK Contract

What every Strike Team writes. The SDK is a thin, stable interface owned by the Core Code Team. Strike Teams import from it — they never touch core internals.

What Strike Teams must provide (the minimum)

Every assistant directory needs exactly one file the build system reads: index.ts. It exports the assistant's definition and, if needed, a handler.

AI ASSISTANT (chat-based) — e.g. Analytics
// src/assistants/analytics/index.ts
import type { AssistantModule }
  from '../../sdk'

const module: AssistantModule = {
  definition: {
    id: 'analytics',
    label: 'Analytics',
    kind: 'ai',
    uiMode: 'chat',
    intro: '...',
    quickActions: [...],
  },
  // No handler needed —
  // core routes to LLM automatically
}

export default module
AI assistants just need a definition. The core handles LLM routing automatically.
TOOL ASSISTANT (custom logic) — e.g. Evergreens
// src/assistants/evergreens/index.ts
import type { AssistantModule }
  from '../../sdk'
import { handler } from './handler'

const module: AssistantModule = {
  definition: {
    id: 'content_table',
    label: 'Evergreens',
    kind: 'tool',
    uiMode: 'tool',
    quickActions: [...],
  },
  handler,  // custom pre-LLM logic
}

export default module
Tool assistants add a handler. The handler interface is unchanged from what exists in core today.

What the handler interface looks like (already exists in core — unchanged)

// src/assistants/evergreens/handler.ts
import type { AssistantHandler, HandlerContext, HandlerResult }
  from '../../sdk'

export const handler: AssistantHandler = {

  // Which actions does this handler own?
  canHandle(assistantId, actionId) {
    return assistantId === 'content_table'
      && (actionId === 'generate-table'
          || actionId === 'add-to-table')
  },

  // What happens when the action runs?
  async handleResponse(ctx: HandlerContext): Promise<HandlerResult> {
    // ... Evergreens scanning logic here
    return { handled: true }
  }
}
This interface already exists in src/core/assistants/handlers/base.ts. The Evergreens handler already implements it. Migration = moving the file and updating the import path. No logic changes.
For new Strike Teams: Copy the analytics/ template → change the ID, label, quickActions → done. If they need custom logic (like Evergreens), they also copy the handler template and implement canHandle + handleResponse. The SDK documentation covers both patterns with examples.
