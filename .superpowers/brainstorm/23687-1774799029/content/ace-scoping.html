How should ACE work for Strike Team members?

When someone from the Evergreens Strike Team logs into the ACE admin, what do they see? This shapes the entire role + permissions model.

A

Scoped view — only their assistant Recommended

A Strike Team member sees only their assistant's settings. Other assistants and global config are hidden. Core Team admins see everything.

CORE TEAM (admin)
Config All Assistants All KBs Design Systems Users
Sees + edits everything
EVERGREENS TEAM (editor)
Evergreens Assistant Evergreens KBs Evergreens Knowledge
Only sees their assistant + its KBs

Pros

  • Clean, focused UX per team
  • No accidental changes to other teams' settings
  • Maps directly to "team ownership"

Cons

  • Requires scoped role model in ACE backend
  • More ACE UI work to implement
B

Full view, read-only outside their scope

Strike Team members see everything but can only edit their own assistant's settings. Other assistants and global config are visible but locked.

EVERGREENS TEAM — sees everything, edits only theirs
✏ Evergreens (editable) 🔒 Analytics (read-only) 🔒 Global Config (read-only) 🔒 Users (hidden)

Pros

  • Teams can see how other assistants are configured (useful for learning)
  • Easier to implement (just disable save buttons)

Cons

  • May expose sensitive config to all teams
  • Cluttered UX — shows things they can't use
C

Keep current model (global roles only) for now

Don't change the ACE role model yet. Focus on the code extensibility architecture first. Add per-assistant ACE scoping in a future phase.

Current roles: admin → manager → editor → reviewer
All global. Any editor can edit any assistant.
Phase 2: add per-assistant role scoping later.

Pros

  • No ACE changes needed in this phase
  • Fastest path to code extensibility

Cons

  • No ownership boundary in the admin
  • Teams can accidentally overwrite each other

My recommendation is A: scoped views per Strike Team. It's the right architecture even if it takes a little more work. Option C is a valid "phase it in" choice if you want to ship the code structure first.
